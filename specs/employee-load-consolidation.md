# Spec for Employee Load Page Consolidation

figma_component (if used): n/a

## Summary

Убрать дублирующий раздел «Загрузка сотрудников» (`/dashboard`) и доработать страницу «Тепловая карта» (`/heatmap`), переименовав её в «Нагрузка сотрудников». Страница получает новый верхний блок — нагрузка по ролям в процентах (аналог блока из удалённого `/dashboard`), а в существующей тепловой карте добавляется поп-ап с детализацией нагрузки по клику на ячейку.

## Functional Requirements

### 1. Удалить раздел «Загрузка сотрудников»

- Удалить файл `app/(app)/dashboard/page.tsx`.
- Убрать пункт «Загрузка сотрудников» из навигации в `components/layout/Sidebar.tsx`.
- Убрать защиту маршрута `/dashboard` из `middleware.ts` (если есть).

### 2. Переименовать «Тепловая карта» → «Нагрузка сотрудников»

- Сайдбар: label `'Тепловая карта'` → `'Нагрузка сотрудников'`.
- Заголовок страницы (`PageHeader`) → `'Нагрузка сотрудников'`.
- Подзаголовок страницы — сохранить или обновить на что-то вроде «Загрузка по ролям и по сотрудникам».

### 3. Добавить блок «Нагрузка по ролям» на страницу `/heatmap`

Новый блок размещается **над** существующей тепловой картой сотрудников.

**Визуал:**
- Идентичен блоку «НАГРУЗКА ПО РОЛЯМ — СЛЕДУЮЩИЕ 6 МЕСЯЦЕВ» из удалённой страницы `/dashboard`, но с горизонтом **12 месяцев** (совпадает с тайм-лайном тепловой карты ниже).
- Заголовок блока: `НАГРУЗКА ПО РОЛЯМ`.
- Для каждой роли — строка баров с процентом загрузки (% от `settings.hoursMonth × кол-во сотрудников в роли`).
- Цвета баров: берутся из `ROLE_COLORS`; при перегрузке (> 100%) — `var(--accent4)`, предупреждение (> 80%) — `var(--accent3)`.
- Tooltip на баре: «Роль: X ч загружено / Y ч норма · Z%».

**Навигация:**
- Блок нагрузки по ролям и блок тепловой карты сотрудников используют **общий** `offset` и общий `PeriodNav` (один переключатель ±1 мес для всей страницы).
- Горизонт обоих блоков — 12 месяцев.

**Фильтр ролей:**
- Существующий фильтр ролей `FilterButtons` над тепловой картой сотрудников фильтрует **только** нижний блок (тепловую карту), верхний блок всегда показывает все роли.

### 4. Поп-ап по клику на ячейку тепловой карты

При клике на ячейку `(сотрудник × месяц)` в нижнем блоке открывается поп-ап (аналогичный поп-апу в разделе «Сроки этапов»).

**Содержимое поп-апа:**
- Заголовок: `{Имя сотрудника} — {Месяц/Год}`.
- Итоговая загрузка: `X ч из Y ч (Z%)`.
- Список вкладов в загрузку, сгруппированных по договору → этапу → задаче:

```
Объект: <название объекта>
  Договор: <название договора> (<тип услуги>)
    Этап 1: <часы> ч
      Задача: <artifact / task из нормы> — <роль> — <часы> ч
      ...
```

**Данные для поп-апа:**
- Перебрать все активные договоры, в чьей команде есть нужный сотрудник.
- Для каждого договора и каждого этапа, попадающего в выбранный месяц (по рабочим дням, аналогично `calcLoad`), посчитать вклад в часы этого сотрудника.
- Показать только те договоры/этапы, у которых вклад > 0 в данный месяц.
- Если данных нет — показать «Нет задач в этом месяце».

**Поп-ап:**
- Использует компонент `Modal` из `components/ui/index.tsx` (без кнопки «Сохранить», только «Закрыть»).
- Открывается кликом на любую ячейку строки сотрудника (включая `hm-0`).

## Possible Edge Cases

- Сотрудник есть в команде договора, но у этапа нет `startDate` — игнорировать этот этап в поп-апе.
- Все нормы для роли дают 0 часов (нет нормативов) — показать строку с 0 ч, не скрывать договор.
- Перегрузка (часов больше нормы) — поп-ап корректно показывает сумму > `hoursMonth`.
- Фильтр ролей скрывает сотрудника из нижнего блока, но верхний блок роли показывает всегда.
- Горизонт 12 месяцев при навигации назад/вперёд — `calcLoad` уже поддерживает `offset`, логика остаётся той же.

## Acceptance Criteria

- [ ] Раздел `/dashboard` удалён, пункт в сайдбаре отсутствует.
- [ ] Страница `/heatmap` называется «Нагрузка сотрудников» в сайдбаре и в заголовке.
- [ ] Вверху страницы есть блок с барами нагрузки по ролям за 12 месяцев.
- [ ] Блок ролей и тепловая карта синхронно переключают период одним `PeriodNav`.
- [ ] Клик на ячейку тепловой карты открывает поп-ап с детализацией по объекту / договору / этапу / задаче.
- [ ] Поп-ап закрывается кнопкой «Закрыть» или кликом вне модального окна.
- [ ] Визуал блока ролей соответствует стилю нижнего блока (те же шрифты, карточки, цвета).

## Open Questions

- Нужно ли сохранять страницу `/dashboard` как редирект на `/heatmap` (для совместимости с закладками)? → Скорее всего нет, маршрут внутренний.
- Показывать ли в поп-апе только задачи с ненулевым вкладом, или все задачи этапа? → Предлагается только ненулевые.
- Нужна ли пагинация/скролл внутри поп-апа если договоров много? → `Modal` уже имеет `max-h-[85vh] overflow-y-auto`, достаточно.
