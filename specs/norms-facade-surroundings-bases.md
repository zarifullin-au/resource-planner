# Spec for Norms Base — Facade Complexity & Surroundings Options

figma_component (if used): —

## Summary

В редакторе норм (поле «База расчёта») добавить два новых варианта базы:
- **«Сложность фасадов»** — значение нормо-часов умножается на `kFacade` объекта
- **«Окружение»** — значение нормо-часов умножается на `kSurroundings` объекта

Это позволяет явно задавать в нормах, какой параметр объекта является базой, а не применять коэффициенты паушально ко всем Визуализаторским нормам с ненулевой базой.

---

## Functional Requirements

### 1. Расширить список баз (`lib/calc.ts`)

Добавить два новых значения в массив `BASES`:
```
'Сложность фасадов'
'Окружение'
```

### 2. UI — выпадающий список в редакторе норм

В компоненте редактирования нормы (поле «База») эти два новых значения появляются в `<select>` наравне с существующими опциями.

### 3. Расчётная логика (`lib/calc.ts` — `calcStageHours`)

Добавить обработку новых баз при вычислении `baseVal`:

| `norm.base` | `baseVal` |
|---|---|
| `'Сложность фасадов'` | `getFacadeK(object.facadeComplexity)` |
| `'Окружение'` | `getSurroundingsK(object.surroundings)` |

Существующие базы (`'Площадь объекта'`, `'Кол-во комнат ...'` и т.д.) — без изменений.

### 4. Убрать паушальное применение kFacade / kSurroundings

**Удалить** блок в `calcStageHours` (и в `getRoleHoursForStage` в slotFinder), который умножал часы Визуализатора на kFacade×kSurroundings по условию `role === 'Визуализатор' && base !== 'Нет'`.

Вместо этого вся логика определяется явно через поле `base` нормы:

```
hours = n_часов × baseVal × kC × kT

где baseVal:
  'Нет'                    → 1
  'Площадь объекта'        → object.area
  'Кол-во комнат основные' → object.roomsMain
  ... (остальные числовые базы без изменений)
  'Сложность фасадов'      → getFacadeK(object.facadeComplexity)
  'Окружение'              → getSurroundingsK(object.surroundings)
```

Таким образом, нормировщик явно контролирует, на что умножаются часы, выбирая базу.

### 5. То же в `lib/slotFinder.ts` (`getRoleHoursForStage`)

Зеркально обновить логику `baseVal` и условие применения коэффициентов.

---

## Possible Edge Cases

- Существующие нормы с базой `'Нет'` или числовыми базами — расчёт не меняется.
- Если объект имеет `facadeComplexity = 'Не выбрано'` → `getFacadeK` возвращает 1, эффект нейтральный.
- Тот же принцип для `surroundings = 'Не выбрано'` → `getSurroundingsK` = 1.

---

## Acceptance Criteria

- [ ] В редакторе норм поле «База» содержит варианты «Сложность фасадов» и «Окружение».
- [ ] Норма с `base = 'Сложность фасадов'` при расчёте использует `kFacade` как `baseVal`.
- [ ] Норма с `base = 'Окружение'` при расчёте использует `kSurroundings` как `baseVal`.
- [ ] Нормы с другими базами работают без изменений.
- [ ] Slot Finder учитывает новые базы корректно.

---

## Open Questions

*Нет. Паушальная логика kFacade×kSurroundings удаляется — управление полностью через поле `base` нормы.*
