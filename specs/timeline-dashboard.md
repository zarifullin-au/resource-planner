# Spec for Timeline Dashboard

figma_component (if used): —

## Summary

Новая страница **«Дашборд»** в разделе «Обзор» (sidebar, маршрут `/timeline`).  
Даёт быстрый ответ на два вопроса:

1. **Что заканчивается** — какие этапы и договора завершаются в ближайшие 6 месяцев.
2. **Когда можно брать новый договор** — в каком месяце у команды появляется достаточно свободной ёмкости, чтобы взять дополнительный объём работ.

Данные берутся из тех же источников, что и остальные страницы (`useAppData()`), без новых API-маршрутов.

---

## Functional Requirements

### Блок 1 — «Завершающиеся этапы и договора»

- Показывает горизонтальный таймлайн на 6 месяцев вперёд от текущего месяца.
- **Единица отображения — этап договора.**  
  Каждый этап рисуется полосой от своей даты начала до даты окончания (`startDate + workingDays → endDate`).
- Полосы сгруппированы по договорам; договор отображается заголовком группы с именем объекта и типом услуги.
- **Цветовая индикация срочности окончания этапа:**
  - Заканчивается в текущем месяце — красный (`var(--accent4)`).
  - Заканчивается в следующем месяце — оранжевый (`#f59a4a`).
  - Заканчивается через 2–3 месяца — жёлтый (`var(--accent3)`).
  - Позже — нейтральный серый / accent.
- Если дата начала этапа раньше видимого окна — полоса обрезается слева и показывается со скошенным краем или другой визуальной меткой «уже идёт».
- Договора и этапы, заканчивающиеся **за пределами 6-месячного окна**, не отображаются.
- Над таймлайном — заголовки месяцев (аналогично `PeriodNav` на других страницах); навигация вперёд/назад по периоду через `PeriodNav`.
- Тултип по наведению на полосу: название этапа, дата начала, дата конца, длительность в р.д.

### Блок 2 — «Окно для нового договора»

- Показывает горизонтальный таймлайн тех же 6 месяцев.
- Для каждого месяца и каждой роли вычисляется **свободный остаток часов** = `(кол-во сотрудников роли × hoursMonth) − суммарная нагрузка роли в месяц` (уже считается в `calcLoad`).
- Строки таймлайна — роли (Тимлид, Дизайнер, Визуализатор и т.д.).
- Ячейка месяца показывает:
  - Свободные часы числом.
  - Цветовую заливку: зелёный (≥ 50 % свободно), жёлтый (20–49 %), красный (< 20 % или перегрузка).
- **Сводная строка «Команда»** над ролями: минимальный процент свободного ресурса среди ролей — это «узкое место» для нового договора.
- Опциональный фильтр по роли (аналог `FilterButtons` на странице загрузки).

## Data & Computation

- **Даты окончания этапов** вычисляются в браузере из `stage.startDate` + `stage.workingDays` календарных рабочих дней.  
  Используется та же логика, что в `calcLoad` / `getMonths`. Отдельного API не нужно.
- **Свободные часы** = данные из `calcLoad()` уже доступны; нужно лишь инвертировать (норма − нагрузка).
- Оба блока используют один и тот же `offset` (`useState`) — навигация по периоду синхронна.

## UI Layout

```
[PageHeader: Дашборд | подзаголовок | PeriodNav]

[Карточка: Завершающиеся этапы и договора]
  [шапка месяцев]
  [строки договоров → этапы в виде полос]

[Карточка: Окно для нового договора]
  [шапка месяцев]
  [строка «Команда» — сводный минимум]
  [строки по ролям — свободные часы / %]
```

Стили — CSS custom properties (`var(--surface)`, `var(--accent)`, и т.д.), карточки класс `card`, типографика `font-head`, аналогично остальным страницам.

## Possible Edge Cases

- **Нет этапов в периоде** — блок 1 показывает пустое состояние «Нет завершающихся этапов в выбранном периоде».
- **Этап без `startDate`** — исключается из отображения; не ломает страницу.
- **Все сотрудники перегружены** — блок 2 показывает все ячейки красными и строку «Команда» с 0 % свободного ресурса.
- **Нет сотрудников / договоров** — оба блока показывают подсказку «Добавьте сотрудников и договоры».
- **Длинное название объекта** — обрезается с `truncate` в заголовке группы.
- **Этап длиннее 6 месяцев** — полоса растягивается на всю видимую часть таймлайна, обрезается по правому краю.

## Acceptance Criteria

- [ ] Страница доступна по маршруту `/timeline` и отображается в боковом меню в разделе «Обзор».
- [ ] Блок 1 корректно отображает все этапы, заканчивающиеся в выбранном 6-месячном окне.
- [ ] Цвет полосы этапа соответствует срочности окончания.
- [ ] Тултип на полосе показывает: название этапа, дата начала, дата конца.
- [ ] Блок 2 показывает свободные часы и % для каждой роли в каждом месяце.
- [ ] Сводная строка «Команда» в блоке 2 отражает минимальный свободный % среди всех ролей.
- [ ] Навигация `PeriodNav` синхронно сдвигает оба блока.
- [ ] Все пустые состояния обработаны и не вызывают ошибок.

## Open Questions

1. **Детализация блока 2** — показывать только роли или также отдельных сотрудников (как на странице загрузки)? Отдельных сотрудников
2. **Интерактивность таймлайна** — нужен ли клик по полосе этапа с открытием карточки договора? да
3. **Пороговые значения свободного ресурса** — 50% / 20% достаточно, или нужны другие? 20%
4. **Учёт выходных** — достаточно ли текущей логики `workingDays`, или нужна поддержка праздников? да, добавить официальные праздники Российской Федерации и возможно дополнительно в разделе "настройки" добавить возможность пользователю вручную добавлять дополнительные праздничные дни на каждый год и учитывать их при просчетах.
