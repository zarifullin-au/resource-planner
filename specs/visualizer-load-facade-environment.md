# Spec for Visualizer Load — Facade Complexity & Environment Coefficients

figma_component (if used): —

## Summary

Добавить два новых параметра объекта, влияющих **только** на расчёт трудозатрат Визуализаторов:

1. **Сложность разнообразия фасадов** — `facadeComplexity`: `Легкий` / `Стандартный` / `Сложный`
2. **Окружение** — `surroundings`: `Город` / `Загород населённый` / `Загород ненаселённый`

Каждый параметр задаёт коэффициент, который перемножается с нормо-часами роли `Визуализатор` при расчёте трудозатрат в `calcStageHours()`.

---

## Functional Requirements

### 1. Модель данных — Object

Добавить два новых поля в таблицу `Object` (Prisma schema):

| Поле | Тип | Значения по умолчанию |
|------|-----|-----------------------|
| `facadeComplexity` | enum `FacadeComplexity` | `Легкий` |
| `surroundings` | enum `Surroundings` | `Загород ненаселённый` |

Новые enum-ы:

```
FacadeComplexity: Легкий | Стандартный | Сложный
Surroundings: Город | Загород_населённый | Загород_ненаселённый
```

### 2. Коэффициенты

**kFacade** — коэффициент сложности фасадов:

| Значение | kFacade |
|----------|---------|
| Легкий | 1 |
| Стандартный | 2 |
| Сложный | 4 |

**kSurroundings** — коэффициент окружения:

| Значение | kSurroundings |
|----------|---------------|
| Город | 4 |
| Загород населённый | 2 |
| Загород ненаселённый | 1 |

### 3. Расчётный pipeline

Коэффициенты применяются **только** в `calcStageHours()` при одновременном выполнении двух условий:
- роль нормы = `Визуализатор`
- поле `base` нормы ≠ `Нет` (т.е. у нормы есть база расчёта)

```
если роль == Визуализатор && norm.base != "Нет":
    часы = базовые_нормо_часы × kC × kT × kFacade × kSurroundings
иначе:
    часы = базовые_нормо_часы × kC × kT   (без изменений)
```

### 4. UI — форма объекта (создание и редактирование)

- Добавить два поля-выпадающих списка в модальное окно объекта (`components/objects/`):
  - **Сложность фасадов**: `Легкий` / `Стандартный` / `Сложный`
  - **Окружение**: `Город` / `Загород населённый` / `Загород ненаселённый`
- Значения по умолчанию: `Легкий` и `Загород ненаселённый` (коэффициенты = 1, поведение не меняется для старых объектов).
- Отображать поля рядом с остальными характеристиками объекта (площадь, сложность и т.п.).

### 5. API

- `POST /api/objects` и `PATCH /api/objects/[id]` принимают и сохраняют новые поля.
- `GET /api/objects` возвращает новые поля в ответе.

### 6. Отображение объекта

- В карточке/строке объекта (страница «Объекты») показывать значения `facadeComplexity` и `surroundings` там, где это уместно.

---

## Possible Edge Cases

- **Старые объекты** (до миграции) получают дефолтные значения (`Легкий` / `Загород ненаселённый`), что даёт `kFacade=1` и `kSurroundings=1` — расчёты не изменяются.
- **Slot Finder** (`lib/slotFinder.ts`) тоже использует `calcStageHours` — убедиться, что новые коэффициенты передаются корректно при поиске слотов.
- Оба параметра влияют только на роль `Визуализатор`; если в команде контракта нет Визуализатора, коэффициенты не имеют эффекта.

---

## Acceptance Criteria

- [ ] Миграция БД добавляет поля `facadeComplexity` и `surroundings` с дефолтными значениями.
- [ ] Форма объекта содержит оба выпадающих поля; значения сохраняются и восстанавливаются при редактировании.
- [ ] `calcStageHours()` применяет `kFacade × kSurroundings` к нормо-часам Визуализатора и не меняет часы других ролей.
- [ ] Существующие объекты с дефолтными значениями дают те же цифры, что и до фичи.
- [ ] Slot Finder учитывает новые коэффициенты при поиске доступных слотов.
