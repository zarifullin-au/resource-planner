# Spec for Object Form — EAP Group for Facade & Environment Fields

figma_component (if used): —

## Summary

В модальном окне создания/редактирования объекта поля «Сложность фасадов» и «Окружение» визуально группируются под заголовком **«Для ЭАП»**. В каждом из этих полей добавляется вариант **«Не выбрано»**, который означает, что коэффициент не применяется (эквивалент = 1).

---

## Functional Requirements

### 1. Визуальная группировка

- Обернуть блок с двумя полями в секцию с заголовком «Для ЭАП».
- Стиль заголовка — небольшой лейбл-сепаратор, аналогично другим группам формы.

### 2. Вариант «Не выбрано»

- В `<select>` для «Сложность фасадов» добавить первым пунктом: `Не выбрано`
- В `<select>` для «Окружение» добавить первым пунктом: `Не выбрано`
- Значение по умолчанию для новых объектов: `Не выбрано` (вместо текущих `Легкий` / `Загород ненаселённый`)

### 3. Поведение расчёта при «Не выбрано»

- Если `facadeComplexity === 'Не выбрано'` → `kFacade = 1` (коэффициент не применяется)
- Если `surroundings === 'Не выбрано'` → `kSurroundings = 1` (коэффициент не применяется)
- Логика эквивалентна текущему поведению дефолтов, но семантически явная.

### 4. Модель данных

- Значения `'Не выбрано'` добавить в массивы `FACADE_COMPLEXITY_TYPES` и `SURROUNDINGS_TYPES` в `lib/calc.ts` первым элементом.
- Тип `FacadeComplexity` и `Surroundings` в `types/index.ts` расширить вариантом `'Не выбрано'`.
- Дефолт в `prisma/schema.prisma` изменить на `"Не выбрано"` для обоих полей.
- Существующие объекты с дефолтом `"Легкий"` / `"Загород ненаселённый"` — коэффициент при них уже = 1, поведение не меняется.

---

## Possible Edge Cases

- **Slot Finder**: draft-объект использует те же defaults — обновить `EMPTY_OBJECT_DRAFT` в `ContractSlotFinder.tsx`.
- **API defaults**: в `route.ts` (POST/PUT) `??` fallback обновить на `'Не выбрано'`.

---

## Acceptance Criteria

- [ ] Форма объекта отображает секцию «Для ЭАП» с двумя полями внутри.
- [ ] Оба select'а содержат «Не выбрано» как первый пункт.
- [ ] Новый объект создаётся со значением «Не выбрано» по умолчанию.
- [ ] При «Не выбрано» расчёт трудозатрат Визуализатора не изменяется (kFacade = kSurroundings = 1).
- [ ] Существующие объекты с ранее сохранёнными значениями работают без изменений.
